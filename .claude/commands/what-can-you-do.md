# /what-can-you-do

Denis sometimes doesn't know what to ask. Show him what's possible.

## Format

Print this menu, then ask "What sounds useful right now?"

```
BUILD & SHIP
  "run the migrations"          — push database changes live
  "deploy to vercel"            — ship to production
  "commit this"                 — save current work to git
  "create a PR"                 — package work for review

FIX & DEBUG
  "the [page] is broken"        — I'll find and fix it
  "this error: [paste it]"      — I'll diagnose and fix
  "why is [page] slow"          — I'll profile and optimize
  "/status"                     — full health check

BUILD FEATURES
  "add [thing] to [page]"       — I'll build it
  "make [page] look like [X]"   — describe or screenshot
  "the [thing] should [behave]" — describe the behavior

UNDERSTAND YOUR APP
  "/explain [anything]"          — plain English explanation
  "what does [file] do"          — I'll translate
  "how many files to change [X]" — I'll estimate scope
  "show me the [page] code"      — I'll walk you through it

STRATEGY & THINKING
  "/think"                       — business/product mode
  "/challenge"                   — poke holes in your idea
  "/decide"                      — log a decision
  "/review-week"                 — weekly review

MANAGE SESSIONS
  "/cost"                        — what did this session cost
  "/compact"                     — free up memory (save $$)
  "/focus"                       — am I being efficient?
  "/start"                       — session standup
```
